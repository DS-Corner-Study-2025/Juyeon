    #node.js 패키지
        하나의 프로젝트에서 node.js 이용해 실행할 때 패키지 형태로 구성함. **패키지는 일종의 관리 단위
        npm 이용해 패키지 생성시 package.json 파일 생성됨.
            package.json -> scripts 항목은 일종의 매크로 기능 지원

    #Node.js 모듈 시스템
        모듈이란? 
        독립적으로 존재하는 프로그램의 일부로 재사용이 가능한 것들 
        모니터나 마우스 같은 컴퓨터의 주변 장치에 비유 -> 모듈은 독립적으로 존재 가능, 다른 프로그램의 부품으로도 활용 할 수 있음

        **자바스크립트에서는 독립된 하나의 파일을 '모듈'이라 부름
        자바스크립트 모듈은 대개 특정 정보를 담은 하나의 객체거나 특정 목적을 지닌 복수의 함수로 구성하는 경우가 많음
        
        모듈 시스템 : 모듈을 사용하는 방법

        **ES 모듈 시스템
        : ECMAScript 모듈 시스템의 약자로 ESM
        Node.js는 기본적으로 CJS모듈 시스템 사용 -> package.json에서 설정을 변경해야 ESM 사용 가능

        ESM에서 불러올 값이 많을때 import * as A from B 형식으로 한번에 불러올 수 있음

        ESM에서는 export 키워드 다음에 default를 붙여 모듈의 기본값으로 내보낼 수 있음. 모듈에서 기본값으로 내보내면 다른 모듈이 이 값을 불러올 때 다른 이름을 붙여도 상관 없음

        **export 키워드 
            const PI = 3.141592; 

            function getArea(radius) { 
            return PI * radius * radius;
            }

            function getCircumference(radius) {
            return 2 * PI * radius;
            }

            export { PI, getArea, getCircumference }; //export 키워드를 변수나 함수 선언 앞에 붙여 사용함. 
                                                    //한번에 여러값 보낼때는 다음과 같이 사용

#4장. 리액트 시작하기
    리액트는 Node.js의 라이브러리 가운데 하나
    페이스북 개발팀이 만들어 2013년 오픈소스로 공개 됨 - 인터랙션 많음

    리액트 특징
        - 컴포넌트 기반의 '유연성' 
            : 모듈화 이용해 중복 코드 삭제/ 공통 사용 코드는 컴포넌트 단위의 모듈로 만들어 사용시 호출
        - 쉽고 간단한 업데이트
            : 웹에서 페이지 업데이트하려면 객체모델 조작 필요, 돔에서 수정 사항 찾아서 수정사항 반영..어려움
            리액트에서는 수정 반영 부분을 수리하기 보다 새거로 바꾸는 방식
        - 빠른 업데이트
            : 업데이트 = 페이지 다시 렌더링
            리액트에서는 버츄얼 돔을 활용해 업데이트를 하고, 이 변경사항을 모았다가 한 번에 실제 돔을 업데이트 함. -> 잦은 업데이트에도 성능 떨어뜨리지 않음

    리액트 앱 만들기
        -  Create React App이라는 Node.js 라이브러리를 이용
        - 리액트 앱을 실행하는 명령어는 package.json의 scripts에 작성되어 있음

    리액트 앱의 동작 원리
        리액트 앱에는 어떻게 접속 ?
        : Create React App으로 만든 리액트 앱에는 웹 서버가 내장되어 있음
        npm run start 명령을 실행하면 내장된 웹 서버가 동작후 자동 접속

        * localhost:3000 주소 의미? 
        - 3000은 포트 번호; 컴퓨터에서 실행되고있는 서버 구분

        리액트 앱의 동작 원리
        : 
        1. localhost:3000으로 접속을 요청하면 public 폴더의 Index.html을 반환합니다.
        2. index.html에는 src 폴더의 index.js와 해당 파일이 가져오는 자바스크립트 파일을 한데 묶어 놓은 bundle.js를 불러옵니다. <script> 태그에서 자동으로 추가합니다.
        3.bundle.js가 실행되어 index.js에서 작성한 코드가 실행됩니다.
        4. Index.js는 ReactDOM.createRoot 메서드로 돔에서 리액트 앱의 루트가 될 요소를 지정합니다.
        5. render 메서드를 사용해 돔의 루트 아래에 자식 컴포넌트를 추가합니다. 결과적으로 App 컴포넌트가 렌더링됩니다.
