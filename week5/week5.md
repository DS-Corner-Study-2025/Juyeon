    2.JSX(자바스크립트 xml)
        -> 컴포넌트를 만들 때 사용하는 문법(js공식 문법은x, 확장 문법)
        표현식: 산술, 문자열, 논리
        **논리 표현식의 결과인 불리언 값은 페이지에 렌더링x,String() 형 변환 함수 사용해 문자열로 바꿔야 렌더링 가능
        원시 자료형에 해당하는 숫자, 문자열, 불리언, null, undefined를 제외한 값 사용시 오류 발생
        ; JSX에서는 객체 자료형을 지원x , 객체 자료형 값 렌더링하려면 프로퍼티 접근 표기법 사용 필요

        규칙:
        닫힘 규칙 - 모든 태그에 적용
        최상위 태그 규칙 - 모든 태그 최상위 태그로 감싸야 함 
            *<React.Fragment>로 다른 태그를 감싸면 최상위 태그를 대체 하는 효과, 렌더링은 x
            <React.Fragment> 대신 빈 태그 ‘<></>’를 사용 가능
        
        조건부 렌더링
        : 조건에 따라 페이지 요소의 모습이나 종류를 다르게 표시하고 싶을 때 사용
            삼항 연산자를 활용한 조건부 렌더링
            조건문을 이용한 조건부 렌더링

        JSX 스타일링
            인라인 스타일링
            : JSX의 인라인 스타일링은 style={{스타일 규칙들}}과 같은 문법으로 작성
             HTML과 다르게 객체를 생성한 다음 각각의 스타일을 프로퍼티 형식으로 작성 / 카멜 표기법 사용
             장점 - 하나의 파일 안에서  UI표현위한 html과 스타일위한 css 규칙 함께 작성 가능
             단점 - 가독성 저하, 불필요한 연산 수행 가능

             스타일 파일 분리

    3.컴포넌트에 값 전달하기
        Props
        -> 부모가 자식 컴포넌트에 전달할 수 있는 단일 '객체' 

        재사용 요소, 내용은 다르지만 구조 동일 -> 컴포넌트 제작 -> 컴포넌트 반복 렌더링 -> 각각의 내용 Props로 전달 // 샌드위치 빵이 컴포넌트 속이 Props 즉 세부사항 의미
        * 컴포넌트의 공통 기능이 아닌 세부 기능을 표현할 때 Props 사용

        Props로 값 전달하기
        -> 전달하는 Props는 단일객체
        
            여러개의 값: 
            구조 분해 할당 (값 사용)
            -> Props로 전달된 값이 여러개일때 점 표기법이아니라 구조 분해 할당사용으로 간편하게

            스프레드 연산자 (값 전달)
            -> 일일이 명시하면 가독성 떨어짐 따라서 Props로 값을 하나의 객체로,,

            기본값 설정
            ->defaultProps를 이용해 기본값 설정해 오류방지

    5.컴포넌트와 상태
        State 이해하기
        : 컴포넌트는 State값에 따라 다른 결과를 렌더링

        State의 기본 사용법
         -useState로 생성
         -set함수로 값 변경
         : 변경값 반영 -> 컴포넌트의 업데이트

        State로 사용자 입력 관리
         - input 태그로 텍스트 입력
         : 텍스트 입력시 onChange 이벤트가 발생해 이벤트 핸들러 handleOnChange 호출
         : input 태그에서 type을 date로 설정-> onChange 이벤트 발생시 이벤트 객체의 e.target.value에 문자열로 이루어진 yyyy-mm-dd 형식의 날짜가 저장 -> 별도의 처리 없이 날짜 형식으로 지정 가능
         - 드롭다운(DropDown)은 option태그와 함께 사용 , 박스와 선택지
         온셥 변경시 onChange 이벤트 발생
         - textarea 태그로 여러줄의 텍스트

        Props와 State

        State와 자식 컴포넌트
        -> 부모의 State 값 변하면 해당 State를 Props로 받은 자식 컴포넌트도 리렌더.
