#p1. 카운터 앱 프로젝트

    1.프로젝트 준비하기

        요구사항 분석하기
        : 프로젝트 구현시 해야할 일
        앱을 어떤 설계와 기능으로 구현할지 살펴보기(요구사항 분석)
        요구사항 분석은 마치 레시피를 점점하는 일과 비슷함. 

        컴포넌트 단위로 생각하기
        : 리액트에서 앱 구현시 컴포넌트 단위로 생각이 필요
        하나의 페이지를 하나의 컴포넌트로 구성하는 것도 가능하나 코드가 복잡해져 프로젝트 관리가 어려움
        -> 재사용이 가능한 수준에서 최대한 잘게 쪼개기
        *재사용기준: 하나의 컴포넌트는 단 하나의 역할만 수행한다.

        리액트 앱 만들기
        : 리액트 앱 생성 방법은 ch5와 동일

    2.UI 구현하기
    : UI -> 사용자 인터페이스, 웹 페이지에서 사용자와 상호작용하는 요소
            요소들의 사용성을 높이기 위해 기능을 추가하기도 하고, 특별한 형태나 색상 등의 스타일을 적용하기도 함.

        Viewer 컴포넌트 만들기
        : 카운트 표시 컴포넌트
        Viewer 컴포넌트를 페이지에 렌더링하기 위해서는 App의 자식으로 배치해야 함.

        Controller 컴포넌트 만들기
        : 카운트를 늘리거나 줄이는 컴포넌트
        Controller 컴포넌트를 페이지에 렌더링하려면 Viewer처럼 App의 자식으로 배치해야 함.

        컴포넌트 스타일링하기
        : src 폴더 App.css에서

    3.기능 구현하기
    
        1.State를 이용해 카운터 기능 구현하기
            이벤트 발생시 컴포넌트 값을 동적으로 렌더링하려면 리액트의 State를 사용해야 함.
            (과정)
            카운트 관리할 state 생성 초기값 0으로 설정
            Controller 컴포넌트의 버튼을 클릭해 현재 state값 을 버튼이 전달하는 값과 계산해 변경
            변경된 State 값을 Viewer 컴포넌트에 전달해 페이지의 카운트 값 업데이트

        2.State는 어떤 컴포넌트에 만들까?
            오답 1: Viewer 컴포넌트
                - Viewer 컴포넌트가 Controller 컴포넌트에 setCount를 전달할 방법이 없다
                5장에서 살펴보았듯이 리액트에서 컴포넌트가 다른 컴포넌트에 데이터를 전달할 때는 Props를 사용하는데, Props는 부모만이 자식에게 전달할 수 있음
                Viewer와 Controller 컴포넌트는 부모-자식 관계가 아니므로 어떠한 값도 전달할 수 없다

            오답 2: Controller 컴포넌트
                - 변경된 State 값을 Viewer 컴포넌트에 전달할 방법이 없다
                State 변수 count를 Viewer 컴포넌트에 전달해야 하는데, Viewer와 Controller컴포넌트는 부모-자식 관계가 아니므로 그렇게 할 수 없음

            정답: App 컴포넌트

            *카운트 기능 구현하려면 State를 App컴포넌트에서 만들어야 함.
            - State 값은 Viewer 컴포넌트, set 함수는 Controller 컴포넌트에 전달해야 하기 때문

            *리액트는 State 값이나 set함수를 여러 컴포넌트에서 사용하는 경우, 이들을 상위 컴포넌트에서 관리합니다. 리액트에서는 이 기능을 다른 말로 ‘State 끌어올리기(State Lifting)’라고 합니다.

        3.리액트답게 설계하기
        : 리액트는 규모가 크고 빠른 웹 어플리케이션을 만들기 좋은 기술
        
        Props 전달방향은 언제나 부모 -> 자식 ; 단방향 데이터 흐름
        -> 데이터 흐름을 이해하기 쉽고 관리하기 좋음
        반면, State 변경 이벤트는 자식->부모 역방향 전달 필요

#ch6. 라이프 사이클과 리액트 개발자 도구
    1. 리액트 컴포넌트와 라이프 사이클
        리액트 컴포넌트의 생애주기 = 라이프 사이클
        라이프 스타일 (3단계)
        : 탄생 - 업데이트 -죽음
         mount(탄생) 처음 렌더링
         update(업데이트) state,props값 변하거나 부모 컴포넌트가 리렌더해 자신도 리렌더 될 때
         unmount(죽음) 더이상 렌더링 x
        
        라이프 사이클을 이용해 컴포넌트 제어하는 것을 라이프 사이클 제어 하고함. 
        리액트 훅의 하나인 함수 useEffect를 이용하면 쉽게 제어 가능

    2. useEffect
    : 함수 useEffect는 어떤 값이 변경될 때마다 특정 코드를 실행하는 리액트 훅 "특정 값 검사"

        하나의 값 검사하기
            *useEffect의 용법
            useEffect(callback, [deps])
                    콜백 함수 의존성 배열

        여러 개의 값 검사하기
        : 배열 요소 중 하나가 변경되어도 useEffect는 콜백 함수를 실행
        
        useEffect로 라이프 사이클 제어하기
        : 3단계 중 업데이트 발생 시 특정 코드 실행
         -> useEffect에서 의존성 배열을 인수로 전달하지 않으면 마운트, 업데이트 시점 모두 콜백 함수를 호출합니다. 그러나 코드처럼 콜백 함수 내부에서 조건문과 Ref 객체로 특정 시점에만 코드를 실행하게 만들 수 있습니다. 즉, 마운트 시점(didMountRef=false)에 호출하면 아무것도 출력하지 않고 함수를 종료하고, 업데이트 시점(didMountRef=true)에 호출하면 문자열을 콘솔에 출력합니다.

        컴포넌트의 마운트 제어하기

        컴포넌트 언마운트 제어하기
            클린업
            : 프로그래밍에서 이 개념은 특정 함수가 실행되고 종료된 후에, 미처 정리하지 못한 사항을 처리하는 일

            클린업을 이용해 컨포넌트 언마운트 제어하기
            :  컴포넌트 언마운트 -> 포넌트가 페이지에서 사라질 때 원하는 코드를 실행

    3. 리액트 개발자 도구
    : useEffect를 이용해 State 값이 바뀌면 콘솔에 그 값을 출력 
    이때 복잡한 리액트 앱을 개발하는 과정에서 State값이 변경될 때마다 useEffect 를 수정하고 console.log로 확인한다면 이는 무척 번거로움
